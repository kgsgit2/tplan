🚗 이동 카테고리 테스트 시나리오

📋 테스트 준비

1
브라우저에서 http://localhost:3000/planner 열기
2
개발자 도구 콘솔 열기 (F12 → Console 탭)
3
콘솔 내용 지우기 (Clear 버튼 또는 Ctrl+L)

🧪 테스트 시나리오 1: 기본 이동 경로 계산

1
좌측 사이드바에서 일반 박스 생성 (카테고리: 관광)
2
박스 제목: 경복궁, 장소명: 경복궁 입력 후 저장
3
좌측 사이드바에서 이동 박스 생성 (카테고리: 이동 선택)
4
이동 박스를 경복궁 박스 아래에 배치
5
또 다른 일반 박스 생성 후 제목: 북촌한옥마을, 장소명: 북촌한옥마을 입력
6
북촌한옥마을 박스를 이동 박스 아래에 배치
7
이동 박스를 클릭하여 모달 열기

✅ 예상 결과

  • 카테고리 선택 UI가 표시되지 않음 (이동 카테고리는 고정)
  • 이동 경로 섹션이 표시됨
  • 이동 수단 선택 버튼 3개 표시 (자동차/대중교통/도보)
  • "경로 계산하기" 버튼이 표시됨
8
"🗺️ 경로 계산하기" 버튼 클릭

✅ 예상 콘솔 로그

🚗 경로 계산 시작
현재 박스: {id: ..., title: "이동", category: "이동", ...}
배치된 박스들: [{...}, {...}, {...}]
정렬된 박스들: [...]
현재 박스 인덱스: 1
이전 박스: {title: "경복궁", placeName: "경복궁", ...}
다음 박스: {title: "북촌한옥마을", placeName: "북촌한옥마을", ...}
📍 출발지: 경복궁
🎯 도착지: 북촌한옥마을
✅ 경로 계산 완료: car - 4.5km, 22분

✅ UI 변경사항

  • 출발지: 경복궁 표시
  • 도착지: 북촌한옥마을 표시
  • 예상 거리와 시간 표시 (예: 4.5km, 22분)
  • 소요시간 필드가 자동으로 업데이트됨

🧪 테스트 시나리오 2: 이동 수단 변경

1
경로 계산이 완료된 상태에서 "🚇 대중교통" 버튼 클릭

✅ 예상 콘솔 로그

이동 수단 변경: 대중교통
🚗 경로 계산 시작
✅ 경로 계산 완료: public - 8.7km, 45분
2
"🚶 도보" 버튼 클릭

✅ 예상 콘솔 로그

이동 수단 변경: 도보
🚗 경로 계산 시작
✅ 경로 계산 완료: walk - 2.1km, 28분

🧪 테스트 시나리오 3: 오류 상황

1
이동 박스만 단독으로 생성 (위아래에 다른 박스 없음)
2
이동 박스 클릭 → "경로 계산하기" 클릭

⚠️ 예상 결과

❌ 배치된 박스가 없음

또는

⚠️ 출발지 또는 도착지를 찾을 수 없음

Alert 메시지: "이동 박스 위아래에 장소 정보가 있는 박스가 필요합니다."

🧪 테스트 시나리오 4: 연속 이동 박스

1
박스 순서: 경복궁 → 이동1 → 이동2 → 북촌한옥마을
2
이동1 박스의 경로 계산

✅ 예상 결과

  • 출발지: 경복궁
  • 도착지: 북촌한옥마을 (이동2를 건너뛰고 다음 장소 찾음)
📌 중요 체크포인트:
  • 이동 카테고리 선택 시 카테고리 드롭다운이 숨겨지는지 확인
  • 경로 계산 시 콘솔에 디버그 로그가 제대로 출력되는지 확인
  • 이동 수단 변경 시 경로가 자동으로 재계산되는지 확인
  • 저장 후 다시 열었을 때 경로 정보가 유지되는지 확인
